# task-04-feed-demotion.md

Use `master.md` as the governing instruction.

## Task
Demote the feed into a secondary aggregated experience.

## Goal
Keep feed useful, but remove feed-overemphasis from the product identity.

## Required Changes
Reposition feed as one or more of:
- 전체 모아보기
- 내가 참여한 공간 모아보기
- 익명 인기글
- 기술 / 정보 하이라이트

Reduce:
- feed-first wording
- feed-first prominence
- social-timeline framing

Keep:
- feed route working
- compatibility with current post cards / ranking logic where practical

## Also Do
Audit for UI copy or labels that still make the app feel like a generic social feed app.
Update only the relevant ones.

## Constraints
- avoid broad unrelated cleanup
- preserve existing routes unless needed
- do not remove useful aggregation behavior

## Validation
- feed still loads
- feed is no longer the main app entry identity
- board/space entry surfaces dominate the shell
