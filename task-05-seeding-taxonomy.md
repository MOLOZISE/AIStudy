# task-05-seeding-taxonomy.md

Use `master.md` as the governing instruction.

## Task
Introduce a sensible initial board/space taxonomy and seed strategy for the new product framing.

## Goal
Make the product usable immediately after the pivot by providing a strong default information architecture.

## Deliver
1. recommended default boards
2. recommended default spaces examples
3. suggested visibility/membership/posting mode per item
4. whether each should allow anonymous posting
5. suggested ordering in sidebar/home

- Q&A / 도움요청
- 정보공유
- 복지 / 문화
- 익명 제안
- 익명 고민

Spaces:
- 프로젝트 공간
- 스터디 공간
- TF 공간
- 관심사 모임

## Constraints
- do not invent complicated enterprise governance
- keep the taxonomy lightweight
- optimize for clarity and immediate usability

## If implementing code
Keep seeding/config additive and easy to revise later.
